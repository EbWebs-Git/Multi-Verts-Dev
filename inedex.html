01 · Architecture

Stack technique

🖥️
Frontend
Un seul fichier index.html contenant HTML, CSS et JavaScript. Aucune dépendance externe, aucun build tool. Fonts via Google Fonts CDN uniquement.
HTML / CSS / JS vanilla
⚡
Backend
Google Apps Script déployé en Web App. Expose un endpoint GET et POST. Pas de serveur à maintenir, hébergé par Google.
Google Apps Script
📊
Base de données
Google Sheets comme datastore. Lecture ligne par ligne, écriture via appendRow et getRange. Modifiable directement sans accès technique.
Google Sheets

Flux de données

1
Chargement initial
Le navigateur charge index.html. Au DOMContentLoaded, loadIdeas() est appelé.
GET /exec → JSON array des débats
2
Soumission d'un débat
Le formulaire construit un tableau row et envoie un POST avec action: "append".
POST /exec → body: {action:"append", row:[...]}
3
Vote
Chaque vote envoie un POST avec action: "update", le numéro de ligne, la colonne (8=likes, 9=dislikes) et la nouvelle valeur.
POST /exec → body: {action:"update", row:N, col:8, val:5}
4
Page À propos
Lecture de la cellule Z2 de Feuille 1 via un paramètre GET.
GET /exec?action=apropos → {apropos: "texte..."}
5
Rafraîchissement
Un setInterval rappelle loadIdeas() toutes les 60 secondes pour maintenir la liste à jour.
setInterval(loadIdeas, 60_000)
02 · API Reference

Endpoints Apps Script

GET /exec Liste tous les débats
Retourne un tableau JSON de tous les débats depuis Feuille 1. La première ligne (headers) est utilisée comme clés d'objet.
Réponse : [{id, titre, description, categorie, auteur, groupe, date, likes, dislikes}, ...]
GET /exec?action=apropos Contenu page À propos
Lit la cellule Z2 de Feuille 1 et retourne son contenu.
Réponse : {"apropos": "texte libre..."}
POST /exec Ajouter un débat
Body JSON avec action: "append" et un tableau row dans l'ordre des colonnes A→I.
Body : {"action":"append","row":["id","titre","desc","cat","auteur","groupe","date",0,0]}
POST /exec Mettre à jour un vote
Body JSON avec action: "update", le numéro de ligne Google Sheet (row), la colonne (col — 8 pour likes, 9 pour dislikes) et la nouvelle valeur (val).
Body : {"action":"update","row":3,"col":8,"val":5}
⚠️ Après chaque modification du code Apps Script, il faut créer une nouvelle version dans Gérer les déploiements. Sauvegarder sans redéployer laisse l'ancienne version active.

04 · Installation

Mise en place rapide

1
Créer le Google Sheet
Nouveau fichier Sheets → onglet nommé exactement Feuille 1 → ligne 1 : id, titre, description, categorie, auteur, groupe, date, likes, dislikes. Copier l'ID depuis l'URL.
2
Apps Script — coller le code
Extensions → Apps Script → remplacer tout le contenu par le code ci-dessous :
const SHEET_ID = 'VOTRE_ID_ICI';

function doGet(e) {
  if (e.parameter.action === 'apropos') {
    const sheet = SpreadsheetApp.openById(SHEET_ID)
      .getSheetByName('Feuille 1');
    const texte = sheet.getRange('Z2').getValue();
    return ContentService
      .createTextOutput(JSON.stringify({ apropos: texte }))
      .setMimeType(ContentService.MimeType.JSON);
  }
  const sheet = SpreadsheetApp.openById(SHEET_ID)
    .getSheetByName('Feuille 1');
  const rows = sheet.getDataRange().getValues();
  const headers = rows[0];
  const data = rows.slice(1).map(r => {
    const obj = {};
    headers.forEach((h, i) => obj[h] = r[i]);
    return obj;
  });
  return ContentService
    .createTextOutput(JSON.stringify(data))
    .setMimeType(ContentService.MimeType.JSON);
}

function doPost(e) {
  const sheet = SpreadsheetApp.openById(SHEET_ID)
    .getSheetByName('Feuille 1');
  const body = JSON.parse(e.postData.contents);
  if (body.action === 'append') sheet.appendRow(body.row);
  if (body.action === 'update')
    sheet.getRange(body.row, body.col).setValue(body.val);
  return ContentService.createTextOutput('ok')
    .setMimeType(ContentService.MimeType.TEXT);
}
3
Déployer en Web App
Déployer → Nouveau déploiement → Type : Application Web → Exécuter en tant que : Moi → Accès : Tout le monde → Déployer. Copier l'URL.
4
Mettre à jour API_URL dans index.html
Chercher const API_URL = et remplacer la valeur par l'URL copiée.
5
Héberger et tester
GitHub Pages, Netlify, Vercel — n'importe quel hébergeur statique. En local : python3 -m http.server 8080 (file:// bloque les fetch).
